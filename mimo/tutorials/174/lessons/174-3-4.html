We just had two conditionals execute code by using nests. Why don't we try the same comparison with the &amp;&amp; operator.

int x = 0;
if (x > -5 &&  x < 5 x == 5)
{
 Console.Write("Message is: ");
 Console.Write("Triggered!");
}
Message is: Triggered!

Not bad eh? A lot more readable than a nested block!

Make sure to try both options to see the conditional in action!
