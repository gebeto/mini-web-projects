When we override a function, the original function becomes hidden. Still, there's a way to access it.

class Animal {
  func makeSound() {
    print("Arf!")
  }
}
class Dog: Animal {
  override func makeSound() {
     super self .makeSound()
    print("Woof!")
  }
}
var myDog = Dog()
myDog.makeSound() Arf!
Woof!

Sweet! With the word super, we can call the superclass's implementation of makeSound().

Ouch! Calling self.makeSound() leads to an endless loop of self-calls.
